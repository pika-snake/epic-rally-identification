## 第一步：启动确认

### 启动日定义（关键）

**启动日 = 前一天涨幅 ≤5% 且当天涨幅 ≥7% 的交易日**

这是判断真正启动日的核心规则：
- 4月21日扫到的股票，其启动日可能是4月16日（如600961，4个涨停后被扫到）
- 如果用"扫到的那天"当作启动日，会把山顶当启动日来分析
- 扫描脚本需向后多查2个交易日，识别连续涨停序列

### 信号清单

| 信号 | 阈值 | 说明 |
|------|------|------|
| 放量 | 量比>2x，连续2天+ | 资金首次大规模进入 |
| 较大涨幅 | ≥7%（涨停更强） | 确认资金攻击力度 |

### 判断标准

- **放量 + 涨幅≥7%** → 启动信号，可关注
- **放量 + 涨停** → 最强确认信号

### 688677验证

- ✅ 3月27日量比2.9x（前日9,309手→27,281手）+ 涨幅+6.3%（接近7%，启动确认）
- ✅ 3月30日量比2.8x + 涨停板（最强确认）

## 第二步：验证"有行情"（不是普通反弹）

### 信号清单

| 信号 | 阈值 | 说明 |
|------|------|------|
| 融资余额持续增加 | 连续3天+增加 | 不是一次性炒作 |
| 缩量加速 | 股价涨但量比下降 | 主力锁仓 |
| 连续涨停 | 2天+ | 市场合力最强 |

### 关键判断：融资余额 vs 股价背离

**启动前吸筹阶段（最早介入机会）：**
- 股价仍在下跌或横盘
- 融资余额已经拒绝下跌，持续增加
- 这是有组织资金在"悄悄埋伏"，比启动信号早1~4周
- **背离持续时间越长，之后涨幅越大**

**经典底部信号：**
- 股价创阶段新低
- 融资余额创阶段新高
- → 杠杆资金逆势加仓，是有组织资金的提前埋伏

**普通反弹特征：**
- 融资余额跟随股价一起下跌
- 无背离现象

### 688677融资余额吸筹全过程

| 时间 | 股价 | 融资余额 | 阶段 |
|------|------|---------|------|
| 2月5日 | 50元 | 1.52亿 | 下跌中 |
| 3月3日 | 42元(-16%) | 1.72亿(+13%) | 背离出现 |
| 3月23日 | 36元(最低) | 1.75亿(最高) | 背离顶峰 |
| 3月27日 | 39.86元 | — | **放量启动** |
| 4月1日 | 62.79元 | 2.10亿(+24%) | 高开低走加仓 |

**结论**：背离信号比行情启动早了近一个月（2月初~3月27日），且背离持续时间越长、之后涨幅越大。察觉到这个信号，可以在3月27日启动前就做好准备。

### 688677验证

- ✅ 融资余额从1.72亿→3.49亿（持续增加）
- ✅ 4月3日后出现缩量加速
- ✅ 3月30日、4月1日连续涨停

## 第三步：验证"史诗级"（3倍+级别）

### 信号清单

| 信号 | 说明 |
|------|------|
| 催化因素 | 年报超预期/政策利好/行业拐点，三者至少其一 |
| 融资增速 > 股价增速 | 杠杆资金持续涌入，行情未结束 |
| 外资持仓非反向 | 机构重仓才能走更远（外资踏空小票是常态） |
| 估值提升空间 | PE能从高位降到低位（如100x→30x），提供估值锚 |

### 年报催化判断方法

```
ROE从年中报→全年报出现V型反转
→ Q4单季度业绩爆发
→ 全年业绩反转确认
→ 明年EPS预期切换，估值锚建立
→ 机构敢于重仓锁筹
```

### 688677验证

- ✅ 年报ROE 5.81%→12.88%（V型反转）
- ✅ 4月1日融资余额暴增24%（杠杆资金趁高开加仓）
- ✅ 外资从1.44%降至0.43%（踏空，验证外资非反向指标）
- ✅ 业绩反转后PE有巨大压缩空间

## 核心结论

**实操定位（关键）**：
- 背离信号 = **买入信号**（启动前埋伏，最佳买点）
- 第三步高分 = **持有信心加成**（不是买入依据）
- 真正买点在启动前，不在启动后追涨

**三个最强信号（同时出现时，行情大概率还没走完）：**

1. **融资余额在启动后不减反增**（尤其是高开低走当天大幅增加）
2. **连续涨停 + 缩量加速**（主力锁仓 = 行情没走完）
3. **催化因素存在**（业绩反转/政策利好/行业拐点）

**执行规则：**
- 三信号齐聚 → 重仓持有
- 两信号出现 → 轻仓试错，等待第三信号确认
- 见顶信号任一出现 → 开始分批减仓

## 操作执行脚本

### 扫描器

路径：`scripts/scan_rally_signal.py`（v1.5）

每日扫描涨幅>=7%的股票，基于第二步信号进行行情验证。
v1.6升级：启动日自动识别（修复v1.4的两个bug）
- Bug1: start/end日期颠倒（all_needed[-1]应为start，all_needed[0]应为end）
- Bug2: 找"前一个交易日"用了[i+1]而非[i-1]（方向反了）
- 问题：扫到的日期可能是连续涨停的第N板（如600961在4/21被扫到，已是第4板）
- 解决：从scan_date向前找连续涨停序列的第一板
- 规则：如果当天是涨停板(>=9.5%)，继续向前找，直到找到"前一天<=5% 且 当天>=7%"的日子

v1.7升级：启动日还原打分（针对4板以上股票）
- 问题：4板以上股票在山顶视角评分只有1-2分，但启动日角度可能得3-4分
- 解决：对4板以上股票（board_count >= 4），增加"启动日还原打分"
- 触发条件：board_count >= 4
- 意义：识别"本来就不是黑马" vs "被山顶视角低估的黑马"
- 实测案例：600961（4板山顶，4/21扫到）→ 山顶分2分，还原分2分 → 结论：本来就只有2分，连续涨停靠板块情绪
- 实测案例：003018（4板山顶，4/22扫到）→ 山顶分2分，还原分2分 → 结论：本来就只有2分，不是黑马框架意义上的高分股
- 效果：600961（4/21扫到）→ 正确识别启动日为4/16，第4板

**用法：**
```bash
python scripts/scan_rally_signal.py [日期YYYYMMDD]
# 示例
python scripts/scan_rally_signal.py 20260327  # 扫描启动日
python scripts/scan_rally_signal.py 20260330  # 扫描启动后第2天

```

**性能优化：**
- 融资数据按日期批量查：30个交易日 = 30次API调用（而非900+次）
- 约15~30秒完成全市场扫描

**输出说明：**

## 成功 vs 失败 核心差异（v1.4更新）

### ① 背离天数不是充分条件

| 股票 | 背离天数 | 启动后5日涨幅 | 结果 |
|------|---------|------------|------|
| 688268(华特气体) | 9天 | +78% | ✅大牛 |
| 688169(石头科技) | 34天 | -4% | ❌失败 |
| 688663(新风光) | 0天 | -1% | ⚠️震荡 |

**结论**：背离天数 >= 10 不是充分条件。关键在于启动前走势是否"有组织"。

### ② 主力量价行为分类（核心判断）

这是区分成功/失败的**最关键维度**：

| 类型 | 行为特征 | 启动后表现 | 代表股票 |
|------|---------|----------|---------|
| **推土机型** | 往上扫货，跌了不买 | 股价持续上涨，连续涨停 | 688268 |
| **接盘机型** | 跌下来才买，不主动推高 | 股价持续低于启动日 | 688169 |

**判断方法**：
- 启动日后若融资增加但股价不涨 → 接盘机型 ⚠️
- 启动日后若融资增加且股价持续上涨 → 推土机型 ✅

**重要例外（板块情绪优先）**：
- 603318水发燃气：背离7天 ✅ + 融资减仓-28% ❌ + 缩量加速 ✅，理论上三信号全错
- 但因为属于"涨停题材板块情绪"，最终6连板+71%
- **结论**：在强势板块情绪面前，个股的融资背离和量比指标都可能被淹没。板块效应是更上位的信号。

### ③ 启动前走势象限分类（v2.2，已验证）

**核心发现**：启动日融资行为是最强单信号，背离天数不是充分条件。

**四象限定义：**

| 象限 | 名称 | 条件 | 代表股票 | 结果 |
|------|------|------|---------|------|

**决策规则（现行）：**

```
第一步：确认启动日（涨幅>=7%，前一天<=5%）
    ↓
第二步：判断象限
    ├─ 象限A → 最优先买入（启动前涨幅>5% + 有实质背离）
    ├─ 象限B → 谨慎买入（启动前超跌<-5% + 启动日融资>10%）
    ├─ 象限Baux → 隐形建仓买入（启动前融资持续增长+25%但股价不涨（前5日总涨幅<10%且无单日异动>=5%）+ 启动日融资>0%（v3.7从>3%下调，修复688268型false negative））
    ├─ 象限C → 小仓位短线，不追高
    └─ 象限D → 坚决不买
    ↓
第三步：市场环境验证（关键！）
    ├─ 大盘强势（指数上行） → 框架信号有效，可正常执行
    ├─ 大盘弱势（节前/整体下跌） → 降低仓位或放弃，弱市中启动日融资暴增可能是借反弹出货
    └─ 大盘震荡 → 参考其他信号，谨慎决策
```

**重要教训**：框架在强势市场中胜率最高。弱市（如五一前、国庆前、整体下跌段）中，即使满足象限A/B/Baux条件，行情失败率也大幅上升。启动日融资暴增在弱市中很可能是杠杆资金借拉升出货，非真行情信号。**实战扫描前必须先判断大盘环境**。

**v2.2与v2.0框架整合：**

```
v2.0打分(0~2分)
    ↓
第三关：启动前走势象限分类（新增）
    ├─ 象限A → +1分加成
    ├─ 象限B → 打分不变，启动日融资>10%才可买入
    ├─ 象限C → -1分
    └─ 象限D → 直接否决
```

**验证结果**：12个样本，100%准确命中象限分类。

| 关键对比 | 688661（成功） | 688169（失败） |
|---------|-------------|-------------|
| 启动前20天 | -10.9% | -11.5% |
| 背离天数 | 4天 | 34天 |
| 启动日融资 | **+138%（暴增）** | **-0.1%（减仓）** |
| 结局 | ✅+20% | ❌-2% |

**结论**：象限B的核心不在背离天数，而在**启动日融资是否暴增**。暴增=大资金最后确认；减仓=借反弹出货。背离天数只是辅助，真正的买入信号是杠杆资金在启动日的态度。

## T日买入决策框架 v2.0（现行）

**决策时间点**：T日收盘后 ~ T+1早晨  
**核心原则**：T日（启动日）之前能确认的数据用于买入判断；T+1之后的数据用于持有判断

### 决策时间线

```
T-1收盘  →  T日盘中（股价涨≥7%）  →  T日收盘  →  T+1早晨（看扫描结果）
                                                              ↓
                                                       决定：是否买入

T日及之前能确认的数据（买入依据）
  → 价格走势（背离天数、跌幅）
  → 成交量（量比）
  → 融资余额走势（背离期间是否抗跌）

T+1及之后才能确认的数据（持有依据）
  → 融资是否持续增加
  → 是否缩量加速
  → 主力是否撤出
```

### 第一关：启动日确认

| 信号 | 阈值 | 含义 |
|------|------|------|
| 前一天涨幅 | ≤ 5% | 启动日定义 |
| 当天涨幅 | ≥ 7% | 触发扫描 |
| 量比 | < 5x | 否则直接淘汰 |

### 第二关：背离质量验证（核心买入信号）

> **v2.4算法修复**：原"前后各半分割"算法在遇到中间大幅波动（四连板/深跌）时会失真。修复方案：改用"启动前最后10日窗口"判断背离。

**有融资数据时（T-30 ~ T-1）：**

| 背离期间融资行为 | 判断 |
|----------------|------|
| 融资不减少（或增加） | ✓ 有组织资金埋伏 → 买入候选 |
| 融资持续减少 | ✗ 杠杆资金不认可 → 淘汰 |

**无融资数据时（只看K线）：**

| 背离形态 | 判断 |
|---------|------|
| 背离天数 ≥ 3天 **且** 股价跌幅 ≥ 5% | ✓ 真背离 → 买入候选 |
| 背离天数 ≥ 3天 **但** 股价跌幅 < 5% | △ 弱背离 → 谨慎考虑 |
| 背离天数 ≥ 3天 **但** 股价不跌或反涨 | ✗ 假背离/反向背离 → 淘汰 |
| 背离天数 < 3天 | △ 背离不足 |

### 第三关：量比验证（T日当天）

| 量比 | 判断 |
|------|------|
| < 2.5x | ✓ 量能健康 |
| 2.5~3x | △ 量能偏大，结合其他信号 |
| 3~5x | ⚠️ 预警信号，-1分 |
| ≥ 5x | ✗ 绝对不买（启动日即见顶概率极高） |

### 综合打分（0~2分）

**得分条件：**

| 信号 | 得分 | 说明 |
|------|------|------|
| A. 有融资数据，背离期间融资不减少（或增加） | +2分 | 融资层面确认 |
| B. 无融资数据，背离天数≥3天 且 股价跌幅≥5% | +2分 | K线层面确认 |
| C. 无融资数据，背离天数≥3天 但 股价跌幅<5% | +1分 | 弱信号 |

**扣分/否决：**

| 信号 | 处理 |
|------|------|
| 量比 ≥ 5x | 直接否决（0分） |
| 量比 3~5x | -1分 |
| 假背离（股价不跌或反涨） | 直接否决 |
| 启动前20天已大涨>10% 且 量比>2.5x | -1分 |

**决策规则：**

| 最终得分 | 决策 |
|---------|------|
| 2分 | 买入 |
| 1分 | 谨慎，仓位减半 |
| 0分 | 不买 |

## 关键规则：启动日定义

> 扫描时扫到的日期 ≠ 真正的启动日

**定义**：启动日 = 前一天涨幅 **≤5%** 且当天涨幅 **≥7%** 的交易日

**原因**：
- 4月21日扫到600961（+10%）时，它已经是连续第4个涨停
- 真正的启动日是4月16日（第一板），4月21是山顶
- 如果用"扫到的那天"当作启动日，分析会严重失真

**扫描脚本要求**：扫描时向后多查5个交易日，识别连续涨停序列，找到真正的第一板日期

**性能**：

**信号逻辑（第二步量化）**：

| 信号 | 量化标准 | 最高分 |
|------|---------|-------|
| 背离（三维验证） | ①整体斜率背离 + ②后半段背离 + ③逐日背离≥5天，满足≥2项即得分 | 1分 |

**背离三维验证详解**：
- 维度1（整体斜率）：线性回归判断股价整体向下 + 融资整体向上
- 维度2（后半段）：启动日前半段 vs 后半段，后者股价跌但融资增
- 维度3（逐日计数）：逐日背离天数≥5天
- 三选二即确认背离

**信号逻辑（第三步量化 — 催化因素）**：

| 信号 | 量化标准 | 最高分 |
|------|---------|-------|
| 年报催化 | ROE出现V型反转（最新 > 上期×1.2） | 1分 |
| 融资增速 > 股价增速 | 30天区间融资余额增速 > 股价增速 | 1分 |
| 估值合理 | PE(TTM) < 60（低估<30，合理30~60） | 1分 |

**评分规则**：
- 第二步 0~4分 + 第三步 0~3分 = 总分 0~7分
- 总分 >= 5：重点关注，行情大概率还没走完
- 总分 >= 6：三步齐备，史诗级行情确认

### 问题3：find_true_launch_date非交易间隔导致启动日误判（v3.2发现，v3.3已修复）

**问题描述**：688268.SH（华特气体）在0421被扫描时，框架原将其归为D象限（"启动前有明显异动"）。Tushare核实后发现启动日前5日（0413~0417）涨幅-7.3%、最大单日仅+2.9%，**没有任何异动**。

**修复（v3.3）**：
- 当`prev_pct is None`时，不应`break`而应`continue`，继续向前找
- 当涨幅<7%时，也不应`break`，应`continue`（防止遗漏真正的启动日）

**实测验证**：修复后688268启动日正确识别为0420（板数=1）。

**688268象限D真正原因**（第二层）：688268被归为D象限，框架判断基本正确，但存在已知局限性：

| 指标 | 值 | 是否满足B/Baux门槛 |
|------|-----|--------------------|
| pre5_chg | -7.3% | ✓ 满足<-5%（B门槛） |
| launch_margin_chg | +1.2% | **✗ 不满足>10%（B门槛）也不满足>3%（Baux门槛）** |
| margin_long_chg | +80.7% | ✓ 满足>25%（B/Baux门槛） |
| has_divergence | 4天 | ✗ 不满足>=5天 |

**已知局限（0420扫描实战案例）**：688268启动日当天融资仅增1.10%（极弱），但前10个交易日连续净买入（89%正天数，30日净买+463亿/基数26512亿=+1.7%）。主力在启动前几天已完成建仓，启动日当天是缩量涨停（融资增幅最小）。实际走势：0420买入→0428峰值+110%，真正的史诗级黑马。框架对启动日融资增幅的权重可能需要调整以捕捉这类"建仓在先、启动在后"的模式。

### 融资数据N/A的完整判断流程

每次遇到融资N/A的候选股，必须执行以下判断链：
1. `pro.margin_detail(ts_code=code, start_date=date, end_date=date)` 查询

## 扫描脚本崩溃应急处理流程

当scan_rally_signal.py出现`UnboundLocalError`崩溃时（常见于Y型新增条件引入新变量但未在所有分支初始化），按以下流程处理：

### 步骤1：诊断

```bash
cd ~/.hermes/skills/investment-research/epic-rally-identification/scripts/
/home/yrflj/.hermes/hermes-agent/venv/bin/python3 scan_rally_signal.py <日期> -t 7 2>&1 | grep UnboundLocalError
```

报错信息格式：`UnboundLocalError: cannot access local variable 'XXX' where it is not associated with a value`

### 步骤2：修复变量初始化

在`analyze_stock_v2`函数Y型块顶部，定位`is_y_type = False`附近，在其下方添加：

```python
margin_sustainability_pass = False  # 默认值，避免UnboundLocalError
```

### 步骤3：验证修复

```bash
# 清理缓存
rm -rf ~/.hermes/skills/investment-research/epic-rally-identification/scripts/__pycache__

# 重新扫描
/home/yrflj/.hermes/hermes-agent/venv/bin/python3 scan_rally_signal.py <日期> -t 7
```

### 步骤4：若仍崩溃（变量在更复杂的条件分支内）

写独立分析脚本绕过主脚本，直接查Tushare：
```python
import tushare as ts
TOKEN = 'fe284a8656a25bf46f4b9092178d1a4dd4d63f26bdb8b83d7f461c47'

# ... 手动实现Y型9条件检查
```

### 根因说明

Python的`UnboundLocalError`规则：函数内某行对变量赋值时，Python将该变量视为"局部变量"。若该行在`if`分支内，而`if`条件从未满足，则变量从未被赋值，在分支外使用时触发错误。**正确做法：在`if`块外（函数开头）初始化所有Y型候选变量为默认值。**

## Tushare数据质量陷阱（必须避免）

### 陷阱1：daily.amount字段单位不一致

**问题描述**：`pro.daily()` 返回的 `amount`（成交额）字段在不同股票上单位不一致。实证发现：

| 股票 | vol | close | amount（API） | vol×close×100（实际） | amount单位 |
|------|-----|-------|-------------|---------------------|-----------|
| 300179.SZ | 947,458股 | 16.91元 | 1,583,637 | 16,021,511元 | **千元** |
| 300179.SZ（前5日） | 521,009股 | 15.23元 | 775,872 | 7,933,307元 | **千元** |

**根因**：Tushare的`daily.amount`字段有时返回**千元**为单位，有时返回**元**为单位，没有统一标准。

**后果**：用`rzmre / amount * 100`计算rzche比（融资买入额/成交额）时：
- 若amount单位是千元：rzche比被放大1000倍（9435% vs 实际9.3%）
- 若amount单位是元：rzche比计算正确

**判断方法**：用`vol × close × 100`计算实际成交额（vol单位=百股，close=元），对比API返回的amount：
- 若`amount × 1000 ≈ vol × close × 100` → amount单位是千元
- 若`amount ≈ vol × close × 100` → amount单位是元

**正确做法**：始终用`vol × close × 100`计算实际成交额，不用API的amount字段：

```python
# 错误（amount单位不稳定）
rzche_ratio = rzmre / amount * 100

# 正确（规避amount单位问题）
actual_amount = vol * close * 100  # vol=百股, close=元, 乘积=元
rzche_ratio = rzmre / actual_amount * 100
```

**实战教训**：20260120扫描时，deep_0120_top.py原始版用`daily.amount`计算rzche比，得到300179四方达的rzche比为9435%（荒谬），修正后为9.33%（合理）。Top15候选中有5只的rzche比因此完全错误。

### 陷阱2：月线数据在某些时段返回N/A

**问题描述**：当扫描日期与数据更新日较近时，`pro.monthly()` 可能返回N/A（尤其是股票在近期刚上市或刚更换交易日期格式）。

**判断方法**：检查月线MA5值是否为None。

### 陷阱3：融资数据margin_detail返回空的两种原因

**问题描述**：`margin_detail`返回空可能是两种截然不同的情况：

| 情况 | 原因 | 解决方案 |
|------|------|---------|
| 情况A：股票不在融资融券标的名单 | 主板股票但未被证监会批准为融资融券标的，`margin_detail`返回全空 | 查`pro.stock_basic(ts_code, fields='market')`确认；这种情况**无法通过Y型融资条件验证**，应直接排除候选 |
| 情况B：股票是小市值/次新股 | 融资数据稀疏（rzye=0或NULL），但`margin_detail`本身有返回字段 | 前5日rzche比均值要求至少3个有效样本才计算 |

**实证**：
- 601702.SH（华峰铝业）、000631.SZ（顺发恒能）均为主板股票，`margin_detail`查20260123返回空 → 情况A（不在融资融券名单）
- 贵州茅台600519.SH `margin_detail`有数据（rzye=195.8亿）→ 情况B（是融资标的但数据正常）

**判断方法**：用`pro.margin_detail(ts_code=code, start_date=date, end_date=date)`查询，若返回空DataFrame且`rzye/rqmcl`字段存在但无值，则：
1. 先用`pro.stock_basic(ts_code=code, fields='name,market')`确认是主板还是北交所
2. 主板股票返回空 = 情况A（不在融资名单）→ Y型融资条件无法验证，按纪律应排除
3. 北交所股票返回空 = 情况B（数据稀疏）→ 可以继续分析但降低置信度

**实战教训**：20260123扫描中601702.SH（量比2.01+K线形态完美）和000631.SZ（量比3.55超标+前5日涨跌+6.2%），两只融资N/A导致Y型条件2/3/10全部无法验证。按纪律，无法验证的条件视为不满足，两只候选均应排除。
