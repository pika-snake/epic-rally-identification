## 第一步：启动确认

### 启动日定义（关键）

**启动日 = 前一天涨幅 ≤5% 且当天涨幅 ≥7% 的交易日**

**前15个交易日窗口计算方法（易错！必须用交易日而非自然日）**：

错误做法：用自然日算窗口（含休市日），导致窗口过大、错误包含启动日前更早期的异动。

正确做法：用`trade_cal`的`pretrade_date`链式往前数15个**交易日**（不包括扫描日当天）。示例：0127的前15个交易日是0106~0126（共15个，元旦假期不计入）。

为什么重要：如果用自然日，0127的窗口会变成1224~0126（22天），会把启动日之前更早期的异动（如1224的涨幅）错误纳入窗口，导致误判为"非新启动"。

这是判断真正启动日的核心规则：
- 4月21日扫到的股票，其启动日可能是4月16日（如600961，4个涨停后被扫到）
- 如果用"扫到的那天"当作启动日，会把山顶当启动日来分析
- 扫描脚本需向后多查2个交易日，识别连续涨停序列

|------|------|------|

| 较大涨幅 | ≥7%（涨停更强） | 确认资金攻击力度 |

## 核心结论

**实操定位（关键）**：
- 背离信号 = **买入信号**（启动前埋伏，最佳买点）
- 第三步高分 = **持有信心加成**（不是买入依据）
- 真正买点在启动前，不在启动后追涨

**三个最强信号（同时出现时，行情大概率还没走完）：**

1. **融资余额在启动后不减反增**（尤其是高开低走当天大幅增加）
2. **连续涨停 + 缩量加速**（主力锁仓 = 行情没走完）
3. **催化因素存在**（业绩反转/政策利好/行业拐点）

**执行规则：**
- 三信号齐聚 → 重仓持有
- 两信号出现 → 轻仓试错，等待第三信号确认
- 见顶信号任一出现 → 开始分批减仓

## T日买入决策框架 v2.0（现行）

**决策时间点**：T日收盘后 ~ T+1早晨  
**核心原则**：T日（启动日）之前能确认的数据用于买入判断；T+1之后的数据用于持有判断

### 决策时间线

```
T-1收盘  →  T日盘中（股价涨≥7%）  →  T日收盘  →  T+1早晨（看扫描结果）
                                                              ↓
                                                       决定：是否买入

T日及之前能确认的数据（买入依据）
  → 价格走势（背离天数、跌幅）
  → 成交量（量比）
  → 融资余额走势（背离期间是否抗跌）

T+1及之后才能确认的数据（持有依据）
  → 融资是否持续增加
  → 是否缩量加速
  → 主力是否撤出
```

### 第一关：启动日确认

| 信号 | 阈值 | 含义 |
|------|------|------|
| 前一天涨幅 | ≤ 5% | 启动日定义 |
| 当天涨幅 | ≥ 7% | 触发扫描 |
| 量比 | < 5x | 否则直接淘汰 |

### 第二关：背离质量验证（核心买入信号）

> **v2.4算法修复**：原"前后各半分割"算法在遇到中间大幅波动（四连板/深跌）时会失真。修复方案：改用"启动前最后10日窗口"判断背离。

**有融资数据时（T-30 ~ T-1）：**

| 背离期间融资行为 | 判断 |
|----------------|------|
| 融资不减少（或增加） | ✓ 有组织资金埋伏 → 买入候选 |
| 融资持续减少 | ✗ 杠杆资金不认可 → 淘汰 |

**无融资数据时（只看K线）：**

| 背离形态 | 判断 |
|---------|------|
| 背离天数 ≥ 3天 **且** 股价跌幅 ≥ 5% | ✓ 真背离 → 买入候选 |
| 背离天数 ≥ 3天 **但** 股价跌幅 < 5% | △ 弱背离 → 谨慎考虑 |
| 背离天数 ≥ 3天 **但** 股价不跌或反涨 | ✗ 假背离/反向背离 → 淘汰 |
| 背离天数 < 3天 | △ 背离不足 |

### 第三关：量比验证（T日当天）

| 量比 | 判断 |
|------|------|
| < 2.5x | ✓ 量能健康 |
| 2.5~3x | △ 量能偏大，结合其他信号 |
| 3~5x | ⚠️ 预警信号，-1分 |
| ≥ 5x | ✗ 绝对不买（启动日即见顶概率极高） |

### 综合打分（0~2分）

**得分条件：**

| 信号 | 得分 | 说明 |
|------|------|------|
| A. 有融资数据，背离期间融资不减少（或增加） | +2分 | 融资层面确认 |
| B. 无融资数据，背离天数≥3天 且 股价跌幅≥5% | +2分 | K线层面确认 |
| C. 无融资数据，背离天数≥3天 但 股价跌幅<5% | +1分 | 弱信号 |

**扣分/否决：**

| 信号 | 处理 |
|------|------|
| 量比 ≥ 5x | 直接否决（0分） |
| 量比 3~5x | -1分 |
| 假背离（股价不跌或反涨） | 直接否决 |
| 启动前20天已大涨>10% 且 量比>2.5x | -1分 |

**决策规则：**

| 最终得分 | 决策 |
|---------|------|
| 2分 | 买入 |
| 1分 | 谨慎，仓位减半 |
| 0分 | 不买 |

## 关键规则：启动日定义

> 扫描时扫到的日期 ≠ 真正的启动日

**定义**：启动日 = 前一天涨幅 **≤5%** 且当天涨幅 **≥7%** 的交易日

**原因**：
- 4月21日扫到600961（+10%）时，它已经是连续第4个涨停
- 真正的启动日是4月16日（第一板），4月21是山顶
- 如果用"扫到的那天"当作启动日，分析会严重失真

**扫描脚本要求**：扫描时向后多查5个交易日，识别连续涨停序列，找到真正的第一板日期

## 附录：脚本说明

**路径**：`scan_rally_signal.py`（与本SKILL.md同目录）

**性能**：
- 融资数据按日期批量查：30个交易日 = 30次API调用（而非数千次）

**信号逻辑（第二步量化）**：

| 信号 | 量化标准 | 最高分 |
|------|---------|-------|
| 背离（三维验证） | ①整体斜率背离 + ②后半段背离 + ③逐日背离≥5天，满足≥2项即得分 | 1分 |

**背离三维验证详解**：
- 维度1（整体斜率）：线性回归判断股价整体向下 + 融资整体向上
- 维度2（后半段）：启动日前半段 vs 后半段，后者股价跌但融资增
- 维度3（逐日计数）：逐日背离天数≥5天
- 三选二即确认背离

**信号逻辑（第三步量化 — 催化因素）**：

| 信号 | 量化标准 | 最高分 |
|------|---------|-------|
| 年报催化 | ROE出现V型反转（最新 > 上期×1.2） | 1分 |
| 融资增速 > 股价增速 | 30天区间融资余额增速 > 股价增速 | 1分 |
| 估值合理 | PE(TTM) < 60（低估<30，合理30~60） | 1分 |

**评分规则**：
- 第二步 0~4分 + 第三步 0~3分 = 总分 0~7分
- 总分 >= 5：重点关注，行情大概率还没走完
- 总分 >= 6：三步齐备，史诗级行情确认

### 问题3：find_true_launch_date非交易间隔导致启动日误判（v3.2发现，v3.3已修复）

**问题描述**：688268.SH（华特气体）在0421被扫描时，框架原将其归为D象限（"启动前有明显异动"）。Tushare核实后发现启动日前5日（0413~0417）涨幅-7.3%、最大单日仅+2.9%，**没有任何异动**。

**根因分析（第一层错误）**：`find_true_launch_date`中，当扫描0420时，0420的"前日"0419因清明假期休市导致`prev_pct=None`，原逻辑错误执行`break`，导致0420被跳过，框架继续向前找到了更早的0409（非真正的连续板启动日）。

**修复（v3.3）**：
- 当`prev_pct is None`时，不应`break`而应`continue`，继续向前找
- 当涨幅<7%时，也不应`break`，应`continue`（防止遗漏真正的启动日）

**实测验证**：修复后688268启动日正确识别为0420（板数=1）。

**688268象限D真正原因**（第二层）：688268被归为D象限，框架判断基本正确，但存在已知局限性：

| 指标 | 值 | 是否满足B/Baux门槛 |
|------|-----|--------------------|
| pre5_chg | -7.3% | ✓ 满足<-5%（B门槛） |
| launch_margin_chg | +1.2% | **✗ 不满足>10%（B门槛）也不满足>3%（Baux门槛）** |
| margin_long_chg | +80.7% | ✓ 满足>25%（B/Baux门槛） |
| has_divergence | 4天 | ✗ 不满足>=5天 |

**已知局限（0420扫描实战案例）**：688268启动日当天融资仅增1.10%（极弱），但前10个交易日连续净买入（89%正天数，30日净买+463亿/基数26512亿=+1.7%）。主力在启动前几天已完成建仓，启动日当天是缩量涨停（融资增幅最小）。实际走势：0420买入→0428峰值+110%，真正的史诗级黑马。框架对启动日融资增幅的权重可能需要调整以捕捉这类"建仓在先、启动在后"的模式。

**框架当前处理**：688268被归D象限是框架的已知盲区。这类"主力提前建仓完毕、启动日只是确认"的黑马，融资信号集中在pre-launch窗口而非启动日当天，框架的Baux/B打分体系对此敏感度不足。0420扫描时688268未被标记为黑马，是框架的false negative。

## Tushare数据质量陷阱（必须避免）

### 陷阱1：daily.amount字段单位不一致

**问题描述**：`pro.daily()` 返回的 `amount`（成交额）字段在不同股票上单位不一致。实证发现：

| 股票 | vol | close | amount（API） | vol×close×100（实际） | amount单位 |
|------|-----|-------|-------------|---------------------|-----------|
| 300179.SZ | 947,458股 | 16.91元 | 1,583,637 | 16,021,511元 | **千元** |
| 300179.SZ（前5日） | 521,009股 | 15.23元 | 775,872 | 7,933,307元 | **千元** |

**根因**：Tushare的`daily.amount`字段有时返回**千元**为单位，有时返回**元**为单位，没有统一标准。

**后果**：用`rzmre / amount * 100`计算rzche比（融资买入额/成交额）时：
- 若amount单位是千元：rzche比被放大1000倍（9435% vs 实际9.3%）
- 若amount单位是元：rzche比计算正确

**判断方法**：用`vol × close × 100`计算实际成交额（vol单位=百股，close=元），对比API返回的amount：
- 若`amount × 1000 ≈ vol × close × 100` → amount单位是千元
- 若`amount ≈ vol × close × 100` → amount单位是元

**正确做法**：始终用`vol × close × 100`计算实际成交额，不用API的amount字段：

```python
# 错误（amount单位不稳定）
rzche_ratio = rzmre / amount * 100

# 正确（规避amount单位问题）
actual_amount = vol * close * 100  # vol=百股, close=元, 乘积=元
rzche_ratio = rzmre / actual_amount * 100
```

**实战教训**：20260120扫描时，deep_0120_top.py原始版用`daily.amount`计算rzche比，得到300179四方达的rzche比为9435%（荒谬），修正后为9.33%（合理）。Top15候选中有5只的rzche比因此完全错误。

### 陷阱2：月线数据在某些时段返回N/A

**问题描述**：当扫描日期与数据更新日较近时，`pro.monthly()` 可能返回N/A（尤其是股票在近期刚上市或刚更换交易日期格式）。

**判断方法**：检查月线MA5值是否为None。

### 陷阱3：融资数据margin_detail返回空的两种原因

**问题描述**：`margin_detail`返回空可能是两种截然不同的情况：

| 情况 | 原因 | 解决方案 |
|------|------|---------|
| 情况A：股票不在融资融券标的名单 | 主板股票但未被证监会批准为融资融券标的，`margin_detail`返回全空 | 查`pro.stock_basic(ts_code, fields='market')`确认；这种情况**无法通过Y型融资条件验证**，应直接排除候选 |
| 情况B：股票是小市值/次新股 | 融资数据稀疏（rzye=0或NULL），但`margin_detail`本身有返回字段 | 前5日rzche比均值要求至少3个有效样本才计算 |

**实证**：
- 601702.SH（华峰铝业）、000631.SZ（顺发恒能）均为主板股票，`margin_detail`查20260123返回空 → 情况A（不在融资融券名单）
- 贵州茅台600519.SH `margin_detail`有数据（rzye=195.8亿）→ 情况B（是融资标的但数据正常）

**判断方法**：用`pro.margin_detail(ts_code=code, start_date=date, end_date=date)`查询，若返回空DataFrame且`rzye/rqmcl`字段存在但无值，则：
1. 先用`pro.stock_basic(ts_code=code, fields='name,market')`确认是主板还是北交所
2. 主板股票返回空 = 情况A（不在融资名单）→ Y型融资条件无法验证，按纪律应排除
3. 北交所股票返回空 = 情况B（数据稀疏）→ 可以继续分析但降低置信度

**实战教训**：20260123扫描中601702.SH（量比2.01+K线形态完美）和000631.SZ（量比3.55超标+前5日涨跌+6.2%），两只融资N/A导致Y型条件2/3/10全部无法验证。按纪律，无法验证的条件视为不满足，两只候选均应排除。

### ③ 日期字符串格式化铁律

**Tushare的`trade_date`是`'YYYYMMDD'`字符串，绝对不能对日期数字做f-string格式化！**

错误示例：
```python
# 错误！会把'20260106'变成'202620260106'
window_strs = [f'2026{d}' for d in window]
df[df['trade_date'].isin(window_strs)]  # 匹配永远失败！
```

正确做法：
```python
# 正确！window列表直接就是['YYYYMMDD']格式
window_strs = window  # 直接用，不用任何格式化
df[df['trade_date'].isin(window_strs)]  # 正常匹配
```

**根因**：`f'2026{d}'`中`d`已经是`'20260106'`字符串，拼出来是`'202620260106'`（26位），而非`'20260106'`（8位）。

### ④ 启动日v2.0定义（必须用交易日而非自然日）

**错误做法**：用自然日算15天（含休市日），导致窗口过大（22天而非15天），错误包含启动日前更早期的异动。

**正确做法**：用`trade_cal`的`pretrade_date`链式往前数15个**交易日**（不包括扫描日当天）。

标准代码模板：
```python
def get_trade_window(scan_date, n=15):
    df_cal = pro.trade_cal(exchange='SSE', start_date='20200101', end_date=scan_date)
    df_cal = df_cal[df_cal['is_open']==1].sort_values('cal_date').reset_index(drop=True)
    window = []
    current = scan_date
    for _ in range(n):
        row = df_cal[df_cal['cal_date'] == current]
        if row.empty: break
        pt = row.iloc[0]['pretrade_date']
        if pd.isna(pt): break
        current = str(int(pt))  # 直接转字符串，不要f-string！
        window.append(current)
    window.reverse()
    return window  # 直接是['YYYYMMDD']格式

# 使用
window = get_trade_window('20260316')
window_data = df[df['trade_date'].isin(window)]  # 不要格式化！
```

**v2.0判定规则**：
- 扫描日前15个交易日内涨幅>=7%的天数 = 0 → 扫描日 = 启动日（有效）
- 扫描日前15个交易日内涨幅>=7%的天数 > 0 → 排除（非启动日）

### ⑦ 扫描结果双表格结构（易漏Y型候选）

**发现过程**：用户观察"小实体天数是关键指标"，我取样14只（7成功+7失败）验证后发现**不是越多越好，而是U形关系**：

| 小实体天数 | 样本数 | 成功率 | T+5均值 | 样本 |
|-----------|--------|--------|---------|------|
| 5~6天 | 4只 | 75% | +9.1% | 天工股份✓/耐普矿机✓/凯赛生物✓/怡亚通✗ |
| 7天 | 2只 | **0%** | +1.65% | 棕榈股份✗/六国化工✗ |
| 9天 | 2只 | **100%** | +10.1% | 中来股份✓/舒华体育✓ |
| 11~12天 | 5只 | 20% | +3.3% | 奥美医疗✗/科林电气✗/*ST纳川✗/北方铜业✓/尤洛卡✗ |

**核心规律**：
- **最优区间：5~9天** → 80%~100%成功率，均值+9.1%~10.1%
- **>=10天反向信号**：长期K线极度安静的股票（阴跌/横盘）突然涨停，**诱多概率高**。12天小实体=0%成功率
- **7天是隐蔽陷阱**：介于好坏之间但实际最差（0%成功）
- **深层原因**：小实体天数反映趋势状态而非安静程度。5~9天+前期有一定涨幅=上升趋势中的加速；>=10天+无涨幅=长期阴跌后的突然异动

**框架规则修正**：小实体>=10天应升级为**反向排除条件**（不满足Y型条件，不推荐买入），而不是现在的>=8天通过。7天股票也应谨慎（成功率0%）。

**月线MA5方向验证结论**：月线上升对T+5预测力有限。下降组均值（+6.77%）反而高于上升组（+5.78%），原因：成功案例中月线下降的（中来+22.7%/北方铜业+15.4%/耐普矿机+37.6%）启动前已经大幅上涨，涨多了月线暂时转空，但这类股票本身是强势股。**月线方向作为参考有价值，但不是决定性指标。**
